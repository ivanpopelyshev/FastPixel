First steps


Welcome to the FastPixel API tutorial section! If you love pixel art as much as I am and you are free for a few minutes: please, take a seat comfortably and let's get started!


Basic HTML and JavaScript knowledge is required! Would be a plus if you had an experience with canvas 2d technology.


Okay, first of all we need to put the canvas tag onto the document body: 


<canvas id="canvas" width="128" height="128"></canvas>

At the second step we have to include an API and the main script file where we going to code.

(Here you can find the latest "pxl.js" file) 


<script src="pxl.js"></script>
<script src="main.js"></script>

Open your "main.js" file and make sure that the current browser is cool: 


if (pxl.isSupported()){
	/*code here further*/
} else{
	alert("not suppoted!");
}

Now you can run your HTML page... No messages? - Good, let's continue!


FastPixel API is synchronous and don't use worker threads. But don't worry, it is fast enough to process any kind of pixels!

It's based on MVC design pattern which is great for creating drawing tools. You can have only one model and few view instances which are listen changes from the same source. One may be used as preview and the other one is for drawing directly, for example.

But we going to talk about model manipulations in future tutorials...


Inside if-body make a reference onto the View class: 


var View = pxl.View;

Create the view instance below and make it active: 


View.activeView = View.create({

	// "Layout" is a model here, so we have to specify our model's size:
	"layoutSize": {
		width: 128,
		height: 128
	},

	// Our canvas:
	"element": document.getElementById("CANVAS")
});

As mentioned before, there may be few view instances, but only one of them can be active at a same time. Also, keep in mind that activeView is a static property of View class.


Prepare four color channels (please note that each channel have to be integer and fit in 0..255 range): 


var r = 255;
var g = 0;
var b = 255;
var a = 255;

Communication between components mostly based on "options" object: 


var options = {
	"start": new pxl.Point(0, 0), //x:y position
	"offset": new pxl.Point(32, 32), //width:height
	"pixel": pxl.RGBA(r, g, b, a) //always make a color using this way! In this example it returns 0xFF00FFFF
};

The Point class is just an object which containt two numbers: {"x": 0, "y": 0}. It helps to orient in 2d space and perfect for storing coordinates.


Finally we can draw our first pixel! 


View.activeView.drawRect(options); //pretty big one :)

Everything should be OK and you have to see the magenta pixel 32x32 at the top of your screen.


My congratulations! You has ben passed the first tutorial and on a one step closer to make a pixel art tool!
